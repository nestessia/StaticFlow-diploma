Развертывание

В этом разделе мы рассмотрим различные способы развертывания сайта, созданного с помощью StaticFlow.

Подготовка к развертыванию

Сборка сайта

staticflow build

Это создаст статические файлы в директории output/.

Проверка сборки

staticflow serve --build-dir output

Варианты хостинга

GitHub Pages

  1. Создайте репозиторий username.github.io
  2. Настройте GitHub Actions:
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.8'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install staticflow

      - name: Build site
        run: staticflow build

      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./output

Netlify

  1. Создайте netlify.toml:
[build]
  command = "staticflow build"
  publish = "output"

[build.environment]
  PYTHON_VERSION = "3.8"
  1. Настройте деплой:
  2. Подключите репозиторий к Netlify
  3. Укажите команду сборки: staticflow build
  4. Укажите директорию публикации: output

Vercel

  1. Создайте vercel.json:
{
  "version": 2,
  "builds": [
    {
      "src": "package.json",
      "use": "@vercel/static-build",
      "config": {
        "buildCommand": "staticflow build",
        "outputDirectory": "output"
      }
    }
  ]
}
  1. Создайте package.json:
{
  "name": "my-staticflow-site",
  "version": "1.0.0",
  "scripts": {
    "build": "staticflow build"
  }
}

Amazon S3

  1. Установите AWS CLI:
pip install awscli
  1. Настройте AWS:
aws configure
  1. Создайте скрипт деплоя:
#!/bin/bash
staticflow build
aws s3 sync output/ s3://your-bucket-name/ --delete

Nginx

  1. Установите Nginx:
sudo apt-get install nginx
  1. Настройте виртуальный хост:
server {
    listen 80;
    server_name your-domain.com;
    root /path/to/your/output;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
  1. Скопируйте файлы:
sudo cp -r output/* /var/www/html/

Настройка домена

DNS настройки

  1. Добавьте A-запись:
your-domain.com.  A  your-server-ip
  1. Добавьте CNAME-запись для www:
www.your-domain.com.  CNAME  your-domain.com.

SSL сертификат

Let's Encrypt

  1. Установите Certbot:
sudo apt-get install certbot python3-certbot-nginx
  1. Получите сертификат:
sudo certbot --nginx -d your-domain.com -d www.your-domain.com

Мониторинг

Логи

  1. Настройте логирование в Nginx:
access_log /var/log/nginx/your-domain.access.log;
error_log /var/log/nginx/your-domain.error.log;
  1. Настройте ротацию логов:
logrotate /etc/logrotate.d/nginx

Метрики

sudo apt-get install prometheus

sudo apt-get install prometheus-node-exporter
  1. Настройте Grafana:
sudo apt-get install grafana

Резервное копирование

Автоматическое резервное копирование

  1. Создайте скрипт:
#!/bin/bash
BACKUP_DIR="/path/to/backups"
SITE_DIR="/path/to/site"
DATE=$(date +%Y-%m-%d)

tar -czf "$BACKUP_DIR/site-$DATE.tar.gz" "$SITE_DIR"
  1. Добавьте в crontab:
0 0 * * * /path/to/backup.sh

Восстановление

tar -xzf site-2024-03-20.tar.gz -C /path/to/restore

gzip on;

  1. Настройте кэширование в Nginx:
location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 30d;
    add_header Cache-Control "public, no-transform";
}

Безопасность

Заголовки безопасности

add_header X-Frame-Options "SAMEORIGIN";
add_header X-XSS-Protection "1; mode=block";
add_header X-Content-Type-Options "nosniff";
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains";

Защита от DDoS

  1. Настройте rate limiting:
limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;

location / {
    limit_req zone=one burst=5;
}

SSL/TLS

  1. Настройте современные шифры:
ssl_protocols TLSv1.2 TLSv1.3;
ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305:DHE-RSA-AES128-GCM-SHA256:DHE-RSA-AES256-GCM-SHA384;

Автоматизация

CI/CD

  1. Настройте GitHub Actions:
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Set up Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.8'

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install staticflow

      - name: Build
        run: staticflow build

      - name: Deploy
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.HOST }}
          username: ${{ secrets.USERNAME }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd /path/to/site
            git pull
            staticflow build
            sudo systemctl reload nginx
